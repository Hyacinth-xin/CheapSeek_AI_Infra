# AEC-C2 赛题说明

## 1. 任务

实现 AEC 虚拟 GPGPU 的主机侧软件栈。提交物为 Runtime 动态库；冲击 Excellent 时还需
提交两个离线策略 Agent。

```text
libaec.so
agents/dma_agent.py
agents/kernel_agent.py
```

本题使用确定性虚拟设备，不使用真实 FPGA、Linux 内核驱动或物理 PCIe 性能。所有性能
分均使用虚拟周期。

## 2. 固定平台

| 项目 | 值 |
|---|---:|
| 设备数量 | 1 |
| 设备内存 | 64 MiB |
| Runtime / Device ABI | 2 / 2 |
| AEC ISA / profile | 2 / 1 |
| DMA 通道 | 2 |
| 最大 threads/block | 1024 |
| 最大参数块 | 64 字节 |
| GEMM M/N/K | `[1,256]` |
| 向量长度 | `[1,1,048,576]` |

运行环境为 little-endian Linux。公共接口使用 C ABI，起始代码使用 C++17。

## 3. 评分模块

| 模块 | 分值 | 内容 |
|---|---:|---|
| Runtime | 30 | 查询、错误、内存、复制、Stream、Event、Launch |
| 计算库 | 30 | GEMM、AXPY、DOT、NRM2 |
| 虚拟 Driver | 20 | Device ABI、双 DMA、注册内存、故障恢复 |
| Agents | 20 | DMA 策略与 Kernel image 选择 |

Runtime 必须导出 `aec_runtime.h` 中的全部符号。尚未实现的高等级功能可以返回
`AEC_ERROR_NOT_SUPPORTED`，但不能缺少符号。

## 4. 计算功能

GEMM 定义为：

```text
C[M,N] = A[M,K] × B[K,N]
```

矩阵均为 dense、row-major、non-transposed、unbatched。支持 FP4 E2M1、两种 FP8、
FP16、BF16、FP32、FP64、INT4、INT8 和 INT32。整数 GEMM 输出 INT32。

向量接口为 FP32 AXPY、DOT 和 NRM2。

## 5. 强制执行路径

成功的 `aecLaunch` 和计算接口必须：

1. 通过 `(kernel_id, dtype, variant)` 解析冻结 image handle；
2. 按规范生成 little-endian 参数块；
3. 保留调用者传入的 grid 和 block；
4. 提交 `AEC_DEVICE_OP_ISA_LAUNCH`；
5. 由受控设备解释并退休 AEC 指令。

不得在 Host 侧直接计算结果后绕过 AEC image。评分会检查 image handle、retired count、
trace digest、trap 和虚拟周期。

固定 bundle 共 34 个 image：Vector Add、AXPY、DOT、NRM2，以及 10 种 dtype 的三个 GEMM
variant。

## 6. Agents

两个 Agent 都是独立进程程序。每次调用从 stdin 读取一个 JSON，并只向 stdout 输出一个
JSON。输出必须满足 `schemas/` 中的约束。

- DMA Agent 选择通道、chunk、queue depth 和 zero-copy。
- Kernel Agent 从给定 candidates 中选择合法 image。

## 7. 等级

| 等级 | 必需能力 |
|---|---|
| Basic | 查询/错误、内存、同步复制、Vector Add、FP32/INT32 GEMM |
| Good | Basic + Stream/Event、异步 DMA、注册内存、全部计算、故障恢复 |
| Excellent | Good + 两个合法 Agent，并取得足够的隐藏性能分 |

具体 requirement、分值和 gate 见 `05_Agent与评分标准.md`。

## 8. 禁止事项

不得修改公共头文件、设备库、Kernel bundle、grader 或测试生成规则。不得提交自定义代码
装载器、自定义 ISA/Trace，或使用 Host compute 代替规定的 AEC 执行路径。

本题不要求真实 FPGA、Linux `.ko`、C1 编译器工具链、完整 B3 微架构模型或任意用户
Kernel image。
