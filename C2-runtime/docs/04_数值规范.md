# 数值规范

## 1. GEMM

```text
C[M,N] = A[M,K] × B[K,N]
```

- 矩阵为 dense、row-major、non-transposed、unbatched。
- `M/N/K` 范围为 `[1,256]`，按递增 k 顺序累加。
- A、B、C 的完整 span 必须分别属于一个 live allocation，且三者不得重叠。
- AXPY 允许 `x == y`；其他向量结果不得与输入重叠。

## 2. 数据格式

| API | A/B storage | accumulator | output |
|---|---|---|---|
| F4 | packed FP4 E2M1 | FP32 | packed FP4 |
| F8 | E4M3FN 或 E5M2 | FP32 | 相同 FP8 格式 |
| F16 | IEEE binary16 | FP32 | binary16 |
| BF16 | bfloat16 | FP32 | bfloat16 |
| F32 | binary32 | FP32 | binary32 |
| F64 | binary64 | FP64 | binary64 |
| I4 | packed signed nibble | 精确整数和 | little-endian INT32，饱和 |
| I8 | signed byte | 精确整数和 | little-endian INT32，饱和 |
| I32 | little-endian INT32 | 精确整数和 | little-endian INT32，饱和 |

整数累加不得发生 signed 64-bit wraparound。可使用 signed 128-bit 或任意精度整数。

packed 类型的低索引元素放在 bits 3:0。元素数为奇数时，最后一字节的高 nibble 必须为 0。
所有多字节值均为 little-endian。

## 3. 浮点转换

- 舍入方式为 round-to-nearest, ties-to-even。
- FP4 和 FP8 finite overflow 饱和到同符号最大有限值。
- FP4 E2M1：bias 1，无 NaN/Inf。
- FP8 E4M3FN：bias 7，`0x7f/0xff` 为 NaN。
- FP8 E5M2：bias 15，支持 IEEE Inf/NaN。
- 任意 NaN 输入产生正 canonical quiet NaN。
- load/conversion 保留 signed zero；精确抵消产生 positive zero。

Canonical positive quiet NaN：

| 格式 | 位型 |
|---|---|
| FP8 E4M3FN | `0x7f` |
| FP8 E5M2 | `0x7e` |
| FP16 | `0x7e00` |
| BF16 | `0x7fc0` |
| FP32 | `0x7fc00000` |
| FP64 | `0x7ff8000000000000` |

## 4. 向量运算

AXPY、DOT、NRM2 均使用 FP32，长度范围为 `[1,1,048,576]`。

- AXPY：`y[i] = alpha*x[i] + y[i]`，乘法和加法分别舍入到 FP32。
- DOT：按 index 递增顺序使用 FP32 accumulator。
- NRM2：按相同顺序累加平方，最后执行 FP32 sqrt。
- DOT 和 NRM2 的结果占一个 FP32。

## 5. 比较规则

| 类型 | 比较方式 |
|---|---|
| packed / integer | bit-exact |
| FP4/FP8/FP16/BF16 | bit-exact |
| FP32 GEMM | `atol=1e-5, rtol=2e-5` |
| FP64 GEMM | `atol=1e-12, rtol=1e-11` |
| FP32 reductions | `atol=2e-5, rtol=5e-5` |

符合本规范的 canonical NaN 在评分中视为相等。

## 6. 测试范围

公开测试覆盖各数据格式的正常计算、代表性 packed 输入、向量归约和常见非法 span。隐藏
测试会增加 NaN/Inf、subnormal、signed zero、舍入边界、饱和、生命周期边界、最大合法
规模和固定 seed 随机输入。隐藏测试不会改变本规范。
