# C1：AEC IR 编译器 - 评分细则

## 总分：100 分

| 类别 | 分值 |
|------|-----:|
| A. 编译与执行正确性 | 50 |
| B. 生成代码效率 | 35 |
| C. 泛化与鲁棒性 | 5 |
| D. Agent 自动优化 | 10 |

---

## A. 编译与执行正确性（50 分）

每个测试用例的评测流程：

1. 参赛者编译器从 PTX 输入生成 AEC 二进制
2. 二进制验证检查格式合规性
3. AEC Golden Model 执行二进制并与参考输出比对正确性
4. 仅正确结果纳入性能评分

正确性在 100 个隐藏测试用例上评测：

| 类别 | 数量 | 重点 |
|------|------|------|
| T1 基础 Lowering | 20 | PTX 解析、基础指令 Lowering |
| T2 控制与标量优化 | 20 | CFG、谓词、DCE、CSE、LICM |
| T3 内存优化 | 20 | 内存访问、合并访问、Shared Memory |
| T4 寄存器与调度 | 20 | 寄存器分配、Spill、DDG、List Scheduling |
| T5 Tensor / GEMM | 20 | TMUL、Tensor Load/Store、Tiling |

## B. 生成代码效率（35 分）

性能由 AEC Cycle Model 的 `total_cycles` 衡量。周期数越低得分越高。

| 类别 | 性能分 |
|------|-------:|
| T1 基础 Lowering | 0 |
| T2 控制与标量优化 | 5 |
| T3 内存优化 | 9 |
| T4 寄存器与调度 | 10 |
| T5 Tensor / GEMM | 11 |

**性能评分原则**：对于每个通过正确性验证的测试用例，周期数与官方基线 O0/O2 编译器比较。分数与所有性能加权用例的几何平均改进成正比。

诊断指标（不直接计分）：
- `instruction_count`
- `spill_count`
- `dual_issue_rate`
- `memory_transactions`
- `stall_cycles`

## C. 泛化与鲁棒性（5 分）

50 个自动生成的变异变体测试鲁棒性。变异包括：

- 参数变化（N、矩阵大小）
- 寄存器重命名
- 基本块重排
- 循环次数变更
- 死代码插入
- 寄存器压力增加
- 数据类型变更（PTX-05：FP16 -> FP4/FP8/BF16/FP32/INT4/INT8/INT32）
- 内存复用模式变更

参赛者不得假定公开测试用例结构固定不变。

## D. Agent 自动优化（10 分）

| 子项 | 分值 |
|------|-----:|
| Agent 性能得分 | 8 |
| 自动化循环完整性 | 2 |

### Agent 性能得分（8 分）

加速比的几何平均：

```text
r_agent = T_default / T_agent
```

其中 `T_default` 为默认编译标志下的周期数，`T_agent` 为 Agent 选择配置下的周期数。

| r_agent | 得分 |
|---------|------|
| < 1.0 | 0 |
| 1.0 - 1.10 | 按比例 |
| 1.10 - 1.25 | 按比例 |
| >= 1.25 | 满分（8 分） |

### 自动化循环完整性（2 分）

Agent 必须：
1. 独立运行
2. 读取性能报告
3. 以调整后的配置重新编译
4. 验证结果
5. 生成最终优化报告

## 8. 隐藏测试精度矩阵（PTX-05）

PTX-05 的隐藏测试覆盖全部 9 种 GEMM 精度：

- FP4 E2M1、FP8 E4M3、FP8 E5M2、FP16、BF16、FP32、INT4、INT8、INT32

以及多种矩阵大小：

- 64 x 64 x 64
- 128 x 128 x 128
- 256 x 128 x 512
- 512 x 512 x 256
- 非 16 倍数边界大小
